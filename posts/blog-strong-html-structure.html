Create a strong HTML page structure

Best practice

The purpose of html is used to structure the raw content of a webpage into meaningful sections.

Below are several points of best practice

  • make the sections syntaxtically meaningful when structuring a web page. This makes the code easy to understand. It also makes it easer for screen readers to navigate through the page. SEO use this as well when ranking pages on the search results
  • Use as little markup as possible: this makes the page more readable and more meaningful (avoid using too much of the generic div elements)
  • Make the elements meaningful: This means choosing an element that best describe the content it contain. (i.e using <p> for paragraphs and <li> for lists)
