Create a robust website directory

This guide is based on research for best practice approaches to creating a website directory. These steps will mke the a directory structure that is easy to navigate and scale.

A Web directory can be broken down into 3 points

  • Folder Hierarchy
  • File name
  • Folder name

All naming is done in lowercase.
- This makes the directory consistent and easier to navigate.
- It also avoids operating systems compatibility and usability issues. (i.e linux treats my-file and MY-File as two different files but windows does not)

Folder Hierarchy:

The root or the start folders of a website typically have the following structure.

  • js or scripts: Include the scripting files that take care of the web page behavior
  • css or styles: Include your CSS files that take care of the web page presentation/look
  • blog or posts or articles: Include the html files that take care of the web page structure
  • media or images or assets: Include the media resources of the website.
  • text or data: Includes the raw content of the web pages before any html structuring
  • index.html The main page
  • about.html The about page
  • logo.png The website logo
  • README.txt A text file that provide description of its directory (It is similar to comments in codes pages). It is best to include one README file inside each root folder.

File name:

After naming and structuring the root folders you would need a naming scheme for the rest of the directory files. The name style you choose does not matter as long as it complies with the following:

  • Make it meaningful: Do not use arbitrary names like "myimage.jpg" or "mypost.html". This makes it hard to know what the content of the file later on and it also affects screen readers when they scan the name.
  • Make it short: Longer names are harder to parse in some software and it also limit understanding. Most naming i saw avarages 25 characters.
  • Don't use spaces: Space signifies the end of a character and can halt the page from rendering properly if at all.
  • Don't use special characters: ∿ ! @ # $ % ^ & * ( ) ` ; < > ? , [ ] { } ‘ “ because these characters have explicit programmatic purposes. with the exception of hyphen and underscore character.
  • If using dates use YYYY-MM-DD or YY-MM-DD format.
  • Be consistent: Whatever naming style you decide to pursue it is important to be consistent throughout. Otherwise everything will fall apart pretty quickly.

Hyphen - or underscore _ ?
While either can be used, using hyphen instead of underscore has many advantages. why? SEO support, visible when hyperlink is rendered in screen, and some SE do not recognize underscore.

  • Use leading zeros: when naming sequential files use leading zeros (i.e file-01, file-001, etc).
