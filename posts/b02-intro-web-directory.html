Intro to static website directory

If you navigate your favorite code hosting platform you will find many different directory structures. Some designed based on personal preference and others are with conventions by a particular framework. The approach mentioned here is by no means the only approach but it takes a lot from the recommended designs.

Directory structure

The root folder of the website should include

- A folder to hold the html files.

- A folder to hold the css files.

- A folder to hold supplimental files. (images, documents, etc...)

- A folder to hold the javascript files.(if you use scripts)

- A folder to hold the text or raw data files before any processing.

- A README.txt or README.md to keep metadata about the directory structure

- An index.html.

- An about.html.

- A logo.

Folder naming

- blog, posts or articles for the html folder.

- css or styles for the css folder.

- media, images or assets for supplimental resources folder.

- js or scripts for the scripts folder (if you are use more than one type of script language then naming it scripts makes more snese)

- texts or data for the plain text folder

It is best to include README file in the root directory and sub-directories.

File nameing

- Make the names meaningful and not broad names like "myimage.jpg" or "mypost.html". You want to be able to have a good idea of the content of the file from the name.

- Make it short as longer names will not get processed properly on some software and technology. (e.g. bluetooth will not process files with long names). A good range would be around 50 characters.

- Avoid spaces!! as they have special use in software. They signal the end of a character and can affect the rendering of pages. Instead use hyphens -, underscore _ or camelCase to seperate words

- Don't use special characters like ∿ ! @ # $ % ^ & * ( ) ` ; < > ? , [ ] { } ‘ “ because these characters have explicit programmatic purpose (with the exception of hyphen and underscore character)

- If using dates use the ISO format YYYY-MM-DD or YY-MM-DD.

- Use leading zeros when naming sequentially. File systems use leading zeros (i.e file-01, file-001, etc) to order content.

The most improtant thing to keep in mind when creating a naming scheme is to be consistent. A naming system will not work if its not followed properly.

A note about hyphens and underscores is that hyphens has SEO support, is more visible on hyperlinks than underscores (as hyperlinks tend to have under_scores) and some search engines do not recognize underscores.

CamelCase names might cause processing issues when ported in different operating systems. (i.e linux OS treat my-file and MY-File as two different files but windows OS treat them as the same file)

Examples

A simple naming style: sunflower-200w-01.png

A more invoved naming style: thisIsA-blog_post-19-02-15.html

I took a lot of inspiration from github's huge collection of repos. For more examples you can check my website directory. or any of the other repositories

Conclusion

Staying consistent when creating a directory system is important to keep it working.
